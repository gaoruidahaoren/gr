# PR 合并前 Spec 对账清单

> 贡献者在提交 PR 前必须完成以下检查项。

---

## 一、功能变更检查

### 1.1 Spec 状态检查

| 检查项 | 操作 | 状态 |
|--------|------|------|
| 新功能是否需要新建 spec？ | 若是，在 `spec/features/planned/` 新建 spec | ☐ |
| 现有功能是否需要更新 spec？ | 若是，更新相关字段 | ☐ |
| 功能完成是否需要转移 spec 状态？ | 若是，从 planned → implemented | ☐ |
| 功能废弃是否需要归档 spec？ | 若是，转移至 archived | ☐ |

### 1.2 Spec 完整性检查

对于每条相关 spec：

| 检查项 | 说明 | 必须 |
|--------|------|------|
| `契约` 字段是否与实现一致？ | 接口定义、行为保证 | 是 |
| `验收标准` 是否全部通过？ | 需提供验收证据 | 是 |
| `实现锚点` 是否已填写？ | 页面、组件、服务路径 | 是 |
| `兼容影响` 是否已评估？ | breaking changes 说明 | 若有变更 |

---

## 二、测试完整性检查

### 2.1 单元测试

| 检查项 | 说明 | 覆盖率要求 |
|--------|------|-----------|
| services/ 测试 | 服务层必须测试 | ≥80% |
| utils/ 测试 | 工具函数必须测试 | ≥90% |
| hooks/ 测试 | 自定义 Hook 必须测试 | ≥80% |

### 2.2 集成测试

| 检查项 | 说明 | 必须 |
|--------|------|------|
| 云函数 Mock 测试 | 每个云函数对应 mock | 是 |
| 页面交互测试 | 核心用户流程 | 是 |

### 2.3 执行命令

```bash
# 必须在本地执行并通过
npm run test          # 运行单元测试
npm run lint          # 运行代码规范检查
npm run type-check    # 运行类型检查
```

## 三、文档完整性检查

### 3.1 代码文档

| 检查项 | 说明 | 必须 |
|--------|------|------|
| 组件 JSDoc 注释 | Props 说明 | 是 |
| 函数 JSDoc 注释 | 参数、返回值说明 | 是 |

### 3.2 Spec 文档

| 检查项 | 说明 | 必须 |
|--------|------|------|
| spec 文件已更新 | 相关 spec 字段完整 | 是 |
| changelog 已记录 | 变更说明 | 是 |

## 四、代码质量检查

### 4.1 静态检查

| 检查项 | 工具 | 通过标准 |
|--------|------|---------|
| ESLint | eslint | 无 error |
| TypeScript | tsc | 无 error |
| Prettier | prettier | 无差异 |

### 4.2 安全检查

| 检查项 | 说明 | 必须 |
